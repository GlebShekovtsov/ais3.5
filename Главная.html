АИС "Учёт сотрудников ВТК"

Является современным решением для автоматизации учёта сотрудников с простым и удобным интерфейсом.

Преимущества

Экономия времени

АИС позволяет существенно сэкономить время при учёте сотрудников учебного заведения.

Не требует настройки

АИС не требует дополнительной настройки для корректной работы.

Не требует долгой подготовки

Работать с АИС можно после короткого обучения.

О разработке

АИС разрабатывалась на объектно-ориентированном языке программирования C# с использование API Windows Forms.
Взаимодействие с базой данных реализовано с помощью запросов на языке SQL. Для администрирования СУБД MySQl используется веб-приложение с открытым кодом phpMyAdmin.

Окно авторизации

Данный элемент был продуман в первую очередь, так как основной замысел АИС заключается в том, что её может использовать только уполномоченный сотрудник, зарегестрированный как администратор.
Меню является адаптивным и имеет приятный дизайн.
Адаптация была реализована с помощью элемента API Windows Forms под названием tablelayoutpanel(см. изображение 1). Была реализована проверка логина и пароля(см. изображение 2) и выборка ФИО вошедшего(см. изображение 3). Программа производит проверку роли(см. изображение 4).

Системное меню

Является основным элеметом управления, имеет возможность открытия всех элементом программы(см. изображение 1). Реализован таймер, который по истечении четырех минут выводит программу в окно авторизации(см. изображение 2). Была реализована возможность самостоятельного входа меню авторизации для смены текущего администратора(см. изображение 3), имя которого отображается в левом верхнем углу.

Добавление, удаление и редактирование сотрудников

Основные функции АИС, позволяет добавлять, удалять и редактировать новых сотрудников в базу данных(см. изображение 1,2,3) c помощью класса Empclass(см. изображение 4). Была реализована возможность очистки текстовых полей(см. изображение 5) и проверки ввода нежелательных символов(см. изображение 6).
Была реализована возможность автоматического определения возраста в зависимости от введенной даты рождения и автоматического заполнения поля, если адрес совпадает с местом прописки(см. изображение 7). Была реализована возможность блокировки поля преподаваемых дисциплин, если выбранная должность не преподаватель(см. изображение 8)
